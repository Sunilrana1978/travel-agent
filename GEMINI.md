# Travel Agent — AI Development Guide (GEMINI.md)

This file provides context to AI coding assistants (Gemini CLI, Claude Code, etc.)
about this project's structure and conventions.

## Project Overview

An AI-powered travel itinerary planner built with Google ADK, deployed to
Vertex AI Agent Engine with a Streamlit web UI.

- **Runtime**: Vertex AI Agent Engine (us-west1)
- **Model**: `gemini-2.5-flash` via Vertex AI (ADC auth)
- **GCP Project**: `travel-agent-502518`
- **CI/CD**: Google Cloud Build → `.cloudbuild/`
- **Package Manager**: `uv` (see `pyproject.toml`)

## Project Structure

```
app/                    # Main application package (agents-cli convention)
├── agent.py            # Root agent + App object (Vertex AI ADC auth)
├── agent_engine_app.py # Production Agent Engine wrapper (TravelAgentApp)
├── agents/
│   └── travel_agent.py # Agent definition + SYSTEM_PROMPT + run_agent()
├── tools/              # ADK tool functions (geocode, weather, places, etc.)
├── models/             # Pydantic data models (Itinerary, Place, etc.)
├── ui/                 # Streamlit web interface
│   ├── app.py          # Main Streamlit app
│   └── export.py       # PDF / Markdown export
└── app_utils/
    ├── telemetry.py    # OpenTelemetry + GCS log setup
    └── typing.py       # Shared TypedDicts (Feedback)

.cloudbuild/
├── pr_checks.yaml      # PR: lint + unit tests (no deploy)
└── deploy.yaml         # Push to staging/main: deploy to Agent Engine

deployment/terraform/   # AUTO-GENERATED by `agents-cli infra` — do not edit manually
tests/                  # pytest unit, integration, and e2e tests
```

## Key Commands

```bash
make install        # Install all deps with uv
make playground     # Local ADK web UI (http://localhost:8501)
make lint           # ruff check
make test           # Unit tests only
make test-all       # All tests including e2e
make deploy-staging # Deploy to staging Agent Engine
make deploy-prod    # Deploy to prod Agent Engine
```

## Agent Architecture

1. **`app/agent.py`** — defines `root_agent` (ADK Agent) and `app` (App wrapper)
2. **`app/agents/travel_agent.py`** — full agent definition: SYSTEM_PROMPT, tools list, run_agent()
3. **`app/agent_engine_app.py`** — `TravelAgentApp(AdkApp)` uploaded to Agent Engine
4. **`app/tools/`** — each file exposes one or more ADK tool functions

## Adding a New Tool

1. Create `app/tools/my_tool.py` with a function decorated for ADK.
2. Export it from `app/tools/__init__.py`.
3. Add it to the `tools=[...]` list in `app/agents/travel_agent.py`.
4. Write a test in `tests/test_tools.py`.

## Auth Model

- **Local dev**: Uses `GOOGLE_API_KEY` in `.env` OR `gcloud auth application-default login`
- **Production (Agent Engine)**: Uses Workload Identity (no keys needed)
- Never commit `.env` — it is in `.gitignore`

## CI/CD Flow

```
PR opened          → Cloud Build runs pr_checks.yaml (lint + tests)
Push to staging    → Cloud Build runs deploy.yaml (_ENV=staging)
Push to main       → Cloud Build runs deploy.yaml (_ENV=prod)
```
